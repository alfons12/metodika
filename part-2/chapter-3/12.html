Document template

Document type represents a document which is generated by the system and can be either stored in electronic form or printed out.

Document can be static or dynamic. The static document has a fixed form and is store somewhere in the system so when user requests such document it is just retrieved from the storage and served to the user. Dynamic document though doesn’t exist before the user requests it. Dynamic document consists of two parts - template and data. Upon request the system gets the template for the particular document, fills it with data and serves it to the user.

Each template both static and dynamic needs to have a unique identificator so it is always clear which exact document is generated when mentioned in the documentation. It is very important to include the business goal and the typical use case - who generates the document and for what reason . This is not needed if the document is linked to a business process as it is easy to find who and when needs the document.

From the technical point of view each data field must be described. This is very similar to elements on the Screen (TBD odkaz) so the business description, conditions for populating and the source of data need to be included.

  • Don’t include any source code for example SQL select statement (TBD pravidlo) and preferably link to the source as it should exist as an artifact - web service, DB table etc.
  • Don’t include business logic such as algorithms and business rules in the document template. They should be represented by separate artifacts and just referenced. Describing how to count the fee to be displayed in the document is almost always a duplication as this logic is used somewhere in the application and should be reffrenced instead.
  • Document is always a consumer of data so all data must come from outside. The only data the document knows about is the text. Everything else must retrieved from somewhere.
  • Don’t forget that evaluating decision conditions also needs data. For example you want to show some text in the document conditionally. This data used to evaluate the condition mus is also not part of the document and needs to beretrieved.
