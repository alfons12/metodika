Service

An application service exposes the functionality of components to their environment. It is is an externally visible unit of behavior, provided by one or more components, exposed through well-defined interfaces, and meaningful to the environment. The service element provides a way to explicitly describe the functionality that components share with each other and the functionality that they make available to the environment. It may require, use, and produce data objects.TBD link

For example a CRM system exposes a ContactsInterface including getContact(id) and createContact(contact) services which enables other applications to manipulate contacts without accessing CRM internal data directly.

These services are neither webservices nor any other technical implementation; they simply describe what automated functionality is provided by the system without specifying the how which supports the principle TBD logic/physical. For example the getContact(id) may be implemented by a webservice GET http://hostname:port/crm/api/contacts/id?type=json

Services can require some input data a can produce some output. The input and output are fixed and are represented by primitive values (string, integer) or data objects.

Examples:

CRM.Contacts.getContact()

  • Service returns details of the contact identified by the id
  • CRM Contacts interface
  • Input
    • id - contact id, integer, required
  • Output
    • Contact - link to data object (name:string, email:string, phone: string, age: integer)

CRM.Contacts.createContact()

  • Service creates contact from the data provided
  • CRM Contacts interface
  • Input
    • ContactInput
      • name:string, optional
      • email:string, optional
      • phone: string, optional
      • birthDate: string, optional
  • Output
    • Contact - create Contact data object including the same information as sent in input

Service specification should include following:

  • Business description - what is the purpose of the service, what value it offers to callers
  • System and Interface - which system and its interface exposes the service
  • Input - primitive value or data object describing the input, specify which arguments are required
  • Output - primitive value or data object describing the output

Except the business description service specification should be generated automatically from the repository and automatically mapped and synchronized with the technical implementation documentation such as how to invoke the service.
