State machine diagram

Sate machine are perfect for describing in what states an entity can be. Entity can be whatever object, during analysis it is mostly a domain entity such Order, Payment, Invoice.

The common mistake analysts do is they describe the states of the entity but don't say in which circumstances the change of state occurs. Not only the states should be described but also the transitions. You need to know what causes the Order change state from NEW to PREPARED.

State transition states: "The Order goes from NEW to PREPARED when an employee finishes preparation of the order and it is ready to be shipped."

Another mistake is not connecting the state machines with the entity whose states it desribes. It should be easy to navigate from the entity to its statemachine and vice versa to navigate to the entity from the state machien.
